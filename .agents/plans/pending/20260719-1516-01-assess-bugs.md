## Proposed changes (ordered, validatable)

| Step | Source finding IDs | Change | Files | Remediation Risk | Validation |
|------|--------------------|--------|-------|------------------|------------|
| 1 | B1 | Decide and document the integer rounding policy. If round-half-up is chosen (recommended for least surprise), implement it with `decimal.Decimal(str(x)).quantize(Decimal(0), rounding=ROUND_HALF_UP)` (which is symmetric: `-2.5 -> -3`). Do NOT use `math.floor(x + 0.5)`: plan-review verified it is WRONG for negative `.5` ties (`-0.5 -> 0`, `-1.5 -> -1`, toward zero, not half-away-from-zero). Alternatively, if banker's rounding is intended, keep it and document it explicitly in API.md/docstrings. Do NOT change silently. Line refs (current): `src/vistab.py:2514` `_fmt_int`, `:2522` `_fmt_comma_int`. | src/vistab.py:2514,2522; docs/API.md | Low | New tests pin the chosen behavior for {0.5,1.5,2.5,3.5,-0.5,-1.5,-2.5} INCLUDING negatives; docstring/API updated; other fixtures unchanged (audit for any fixture exercising a `.5` value). |
| 2 | B2 | On a themes.json that exists but fails to parse, emit a one-line WARNING to stderr (path + reason) instead of silent `pass`; still continue without the custom themes. | src/vistab.py:3762-3766 | Low | A deliberately-corrupt themes.json produces a stderr warning naming the file; a valid one still loads silently; startup never crashes. |

Confirm this does not change how legitimate string "None" data renders. | src/vistab.py `_fmt_*`/`_str` None handling | Low | Test: `None` in each numeric code column -> empty (or placeholder); literal string "None" in a `t` column -> unchanged; no crash. |

## Open questions

- B1 rounding policy: RESOLVED (human, 2026-07-19 plan-review) -> **round half-up**. Integer
  codes (`i`/`I`) will use symmetric round-half-away-from-zero (`2.5 -> 3`, `-2.5 -> -3`) via
  `decimal.Decimal(str(x)).quantize(Decimal(0), rounding=ROUND_HALF_UP)`. This is a user-visible
  behavior change to `.5` output; it ships in the next version (CHANGELOG `[Unreleased]` Fixed,
  since it corrects surprising undocumented behavior) and its tests must cover negatives.
- B3 None rendering: RESOLVED (human, 2026-07-19 plan-review) -> **empty cell**. `None` in a
  numeric-code column renders as an empty string; no configurable placeholder / new API is added
  (avoids scope creep). A literal string `"None"` in a `t` (text) column is unaffected.

- Cross-plan (added by plan-review 2026-07-19): the tests IPD
  (`.agents/plans/pending/20260719-1530-01-assess-tests.md`, finding T1) will add a
  characterization test pinning integer `.5`-boundary rounding. B1 here CHANGES that behavior (if
  round-half-up is chosen). Sequence so B1 lands first (or together): the T1 pinning test must be
  written to B1's CHOSEN behavior, never pin banker's rounding and then break it. If B1 is
  approved, execute B1's own rounding tests and T1's pin as one consistent change.
