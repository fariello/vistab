### Note from probing T5 (RESOLVED via a separate bug IPD)

A live probe of the degenerate shapes found that `Vistab().draw()` on a completely empty table
(no header, no rows, data `[]`) returned **`None`**, not a string, breaking naive callers
(`print`, `.splitlines()`, concatenation). That was a latent BUG, so it was filed and
plan-reviewed as a SEPARATE bug IPD: `.agents/plans/pending/20260719-1916-01-empty-table-draw-
returns-none.md` (finding E1, reviewed, GO pending approval), which fixes `draw()` to return
`""` (matching the documented `-> str` contract and `stream()` -> `[]`). SEQUENCING: that bug IPD
executes first (or together); then T5 Step 5d pins the CORRECTED `Vistab().draw() == ""`, never
`None`. To avoid duplication, the single empty-table assertion may live in EITHER plan's tests but
not both (owner: whichever executes first; the other cross-references it). Confirmed-good
degenerate renders from the probe (safe to pin as-is): 1x1 header+empty, 1x1 no-header word,
`['']`/`[None]` one-cell box, 1-col 2-row, colspan-across-two no-header, and multi-line 1x1.
Key distinction locked by both plans: truly-empty `[]` -> `""`; a present one-empty-cell
structure (`['']`, `[None]`, `set_header([''])`) -> a drawn one-cell box.
