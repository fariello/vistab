### Note from probing T5 (RESOLVED and EXECUTED via a separate bug IPD)

A live probe of the degenerate shapes found that `Vistab().draw()` on a completely empty table
(no header, no rows, data `[]`) returned **`None`**, not a string, breaking naive callers
(`print`, `.splitlines()`, concatenation). That was a latent BUG. It was filed, plan-reviewed, and
EXECUTED as a separate bug IPD: `.agents/plans/executed/20260719-1916-01-empty-table-draw-returns-
none.md` (finding E1, commit `389aca2`); `draw()` now returns `""` (matching the documented
`-> str` contract and `stream()` -> `[]`), with `tests/test_vistab.py::TestEmptyTableDraw` already
pinning `Vistab().draw() == ""` AND the `['']`/`[None]`-still-draws-a-box distinction. Therefore
Step 5 must NOT re-add those assertions; it references `TestEmptyTableDraw` and covers only the
remaining degenerate shapes. Confirmed-good
degenerate renders from the probe (safe to pin as-is): 1x1 header+empty, 1x1 no-header word,
`['']`/`[None]` one-cell box, 1-col 2-row, colspan-across-two no-header, and multi-line 1x1.
Key distinction locked by both plans: truly-empty `[]` -> `""`; a present one-empty-cell
structure (`['']`, `[None]`, `set_header([''])`) -> a drawn one-cell box.
